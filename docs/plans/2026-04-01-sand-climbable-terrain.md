# Sand Climbable Terrain Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Make mouse-dropped sand act as climbable terrain so the player's Y position rises to stand on top of any sand pile.

**Architecture:** Two targeted edits to `lib/sand-wizard/player.ts`. First, extend `findSurface` to scan from `y=0` downward so it detects sand piles that have grown above the player's current feet level. Second, change the wall kill check to only fire on solid rock (cell type `2`), not sand (cell type `1`).

**Tech Stack:** TypeScript, Next.js (no test framework — validate with `pnpm build` + browser)

---

### Task 1: Fix `findSurface` to scan from top of grid

**Files:**
- Modify: `lib/sand-wizard/player.ts:55`

**Step 1: Edit `findSurface`**

In `lib/sand-wizard/player.ts`, change line 55 from:

```ts
  for (let y = Math.ceil(player.y); y < gridHeight; y++) {
```

to:

```ts
  for (let y = 0; y < gridHeight; y++) {
```

This makes the scan find the topmost occupied cell in the player's column rather than only looking below the player's current feet.

**Step 2: Verify TypeScript compiles**

```bash
pnpm build
```

Expected: build succeeds with no new errors.

**Step 3: Commit**

```bash
git add lib/sand-wizard/player.ts
git commit -m "fix: findSurface scans from top of grid to detect sand piles above player"
```

### Task 2: Change wall kill to rock-only

**Files:**
- Modify: `lib/sand-wizard/player.ts:32`

**Step 1: Edit wall check**

In `lib/sand-wizard/player.ts`, change line 32 from:

```ts
  if (getCell(state.grid, frontX, chestY, state.gridWidth) !== 0) {
```

to:

```ts
  if (getCell(state.grid, frontX, chestY, state.gridWidth) === 2) {
```

Only solid rock (cell type `2`) kills the player. Sand (cell type `1`) is handled by the surface elevation from Task 1.

**Step 2: Verify TypeScript compiles**

```bash
pnpm build
```

Expected: build succeeds with no new errors.

**Step 3: Manual verification in browser**

Start the dev server:

```bash
pnpm dev
```

Navigate to `/games/sand-wizard`. Start a game, then:

1. Left-click in front of the player (slightly to the right, at ground level) to drop a sand pile 3–4 cells tall. Verify the player rides up over it without dying.
2. Build a tall sand pile (10+ cells) directly in the player's path. Verify the player climbs to the top.
3. Verify the player still dies when walking into a rock wall (the solid rock columns generated by `worldgen`).
4. Verify jumping and ducking still work correctly on sand surfaces.

**Step 4: Commit**

```bash

git commit -m "fix: sand walls no longer kill player — only rock does"
```
